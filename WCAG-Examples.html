WCAG Success Criteria Examples

Comprehensive code examples for Web Content Accessibility Guidelines

üìã Quick Summary

Requirement: All non-text content must have a text alternative that serves the equivalent purpose.

  • Images need descriptive alt text
  • Decorative images should have empty alt=""
  • Complex images need detailed descriptions
  • SVGs need accessible names and roles
  • Charts and graphs need text summaries
  • CAPTCHAs need alternative methods

1. Basic Images

‚ùå Bad Example - Missing alt attribute
<img src="logo.png">

Issue: Screen readers cannot describe the image

‚úÖ Good Example - Descriptive alt text
<img src="logo.png" alt="Company Name - Innovative Solutions">

Why: Provides meaningful description for screen reader users

2. Decorative Images

‚ùå Bad Example - Redundant description
<img src="decorative-line.png" alt="decorative line">

Issue: Adds unnecessary noise for screen reader users

‚úÖ Good Example - Empty alt for decorative
<img src="decorative-line.png" alt="">

Why: Screen readers skip decorative images

3. Complex Images & Charts

‚ùå Bad Example - Insufficient description
<img src="sales-chart.png" alt="chart">

Issue: Doesn't convey the data or meaning

‚úÖ Good Example - Detailed description
<figure>
    <img src="sales-chart.png" 
         alt="Bar chart showing quarterly sales"
         aria-describedby="chart-desc">
    <figcaption id="chart-desc">
        Sales increased from $50k in Q1 to $120k in Q4, 
        with steady growth each quarter. Q4 showed the 
        highest increase at 40% over Q3.
    </figcaption>
</figure>

Why: Provides complete data understanding

4. SVG Graphics

‚ùå Bad Example - No description on SVG
<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" fill="blue" />
</svg>

Issue: Screen readers cannot interpret the graphic

‚úÖ Good Example - Accessible SVG
<svg width="100" height="100" 
     role="img" 
     aria-labelledby="svg-title">
    <title id="svg-title">Blue circle icon</title>
    <circle cx="50" cy="50" r="40" fill="blue" />
</svg>

Why: Provides accessible name for the graphic

5. Image Maps

‚ùå Bad Example - Missing area descriptions
<img src="world-map.png" alt="World Map" usemap="#worldmap">
<map name="worldmap">
    <area shape="rect" coords="0,0,100,100" href="asia.html">
    <area shape="rect" coords="100,0,200,100" href="europe.html">
</map>

Issue: Clickable areas have no descriptions

‚úÖ Good Example - Described areas
<img src="world-map.png" alt="Interactive World Map" usemap="#worldmap">
<map name="worldmap">
    <area shape="rect" coords="0,0,100,100" 
          href="asia.html" alt="Asia region">
    <area shape="rect" coords="100,0,200,100" 
          href="europe.html" alt="Europe region">
</map>

Why: Each clickable area is identified

6. CAPTCHA

‚ùå Bad Example - No alternative
<img src="captcha.png" alt="CAPTCHA">
<input type="text" placeholder="Enter code">

Issue: Visually impaired users cannot complete verification

‚úÖ Good Example - Audio alternative
<div>
    <img src="captcha.png" alt="Visual verification code">
    <button type="button" aria-label="Play audio CAPTCHA">
        üîä Audio Alternative
    </button>
    <input type="text" 
           aria-label="Enter verification code">
</div>

Why: Provides alternative verification method

7. Functional Images (Buttons, Links)

‚ùå Bad Example - Generic alt text
<a href="contact.html">
    <img src="icon.png" alt="icon">
</a>

Issue: Doesn't describe the action or destination

‚úÖ Good Example - Descriptive function
<a href="contact.html">
    <img src="mail-icon.png" alt="Contact Us">
</a>

Why: Clearly indicates the link purpose

8. CSS Background Images

‚ùå Bad Example - Important content in CSS
<div style="background: url('announcement.png')">
</div>

Issue: Content not accessible to screen readers

‚úÖ Good Example - Text alternative provided
<div style="background: url('announcement.png')" 
     role="img" 
     aria-label="Special offer: 50% off all items this weekend">
</div>

Why: Screen readers can access the information

‚úÖ Implementation Checklist

  • ‚úì All <img> tags have alt attributes
  • ‚úì Alt text describes image purpose, not just appearance
  • ‚úì Decorative images use alt=""
  • ‚úì Complex images have detailed descriptions
  • ‚úì SVGs include <title> and role="img"
  • ‚úì Image map areas have alt text
  • ‚úì CAPTCHAs offer audio or other alternatives
  • ‚úì Icon buttons have descriptive labels
  • ‚úì Charts include data tables or summaries

üîç Testing Tips

  • Turn off images and verify all information is still available
  • Use a screen reader (NVDA, JAWS, or VoiceOver)
  • Check with browser extensions (WAVE, axe DevTools)
  • Validate that alt text is meaningful, not just present
  • Ensure decorative images don't clutter screen reader output
