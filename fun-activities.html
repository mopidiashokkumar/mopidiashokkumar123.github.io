ARIA Screen Reader Attributes

Live, interactive demo of three key ARIA features — with real DOM updates
so NVDA announces them exactly as shown.

aria-atomic role="log" role="marquee"
1

aria-atomic

What it does: Controls how much of a live region NVDA reads aloud when any part of it changes.

aria-atomic="true" → NVDA reads the entire region every time, even if only one word changed.
aria-atomic="false" → NVDA reads only the part that changed, not the full region.

Real-world example: A scoreboard shows "Team A — 3 : 1 — Team B". If atomic is true, NVDA says "Team A 3 colon 1 Team B" — the whole thing. If atomic is false, NVDA says only "3" — just the number that changed.

🎧 atomic="true" → NVDA: "Score: 4 out of 10" (full region)
🎧 atomic="false" → NVDA: "4" (only the changed span)
HTML — aria-atomic
<!-- atomic="true" → reads the WHOLE region -->
<div aria-live="polite"
     aria-atomic="true">
  Score: <span id="s1">0</span> out of 10
</div>
<!-- NVDA says: "Score: 4 out of 10" -->

<!-- atomic="false" → reads ONLY the changed span -->
<div aria-live="polite"
     aria-atomic="false">
  Score: <span id="s2">0</span> out of 10
</div>
<!-- NVDA says: "4"  (only the span changed) -->

role="log"

What it does: Marks a region as a sequential, append-only log. Think: chat messages, server console, activity feed, error log.

It implicitly sets aria-live="polite" and aria-atomic="false" — you do NOT need to write those yourself. NVDA will announce each new item that is added to the list, after the user finishes reading what they're on.

Important rule: Only append items — don't replace or reorder existing ones, or NVDA gets confused.

🎧 User is reading something → you append "Alice: Hello!"
→ NVDA finishes current sentence, then says: "Alice: Hello!"

🎧 Next message appended: "Bob: Hi there!"
→ NVDA says: "Bob: Hi there!" ← only the new entry
HTML — role="log"
<!-- role="log" implicitly gives you:
     aria-live="polite"
     aria-atomic="false"
     aria-relevant="additions text"
     You don't need to write those! -->

<ul role="log"
    aria-label="Chat messages">
  <li>Alice: Hello!</li>
  <li>Bob: Hi there</li>
</ul>

/* JS — only ever APPEND, never replace */
const log = document.querySelector('[role="log"]');
const li  = document.createElement('li');
li.textContent = 'Carol: Just joined!';
log.appendChild(li); // ← append only!

role="marquee"

What it does: Marks a region as non-critical, continuously changing content — like a news ticker, stock banner, or rotating carousel text.

It implicitly sets aria-live="off" — meaning NVDA will NOT automatically announce any updates. The user can still manually navigate to this element and read it, but it will never interrupt or auto-announce.

Why? Tickers change constantly. If NVDA announced every update, it would be an endless, disruptive stream of speech. So role="marquee" keeps it silent by default.

🎧 Ticker updates every 3 seconds → NVDA says: nothing (silent)
🎧 User presses arrow key to navigate here → NVDA reads: "News ticker: Breaking — Market up 2%..."
📌 Updates are accessible but not intrusive
HTML — role="marquee"
<!-- role="marquee" implicitly gives you:
     aria-live="off"   ← NVDA STAYS SILENT on updates!
     aria-atomic="false"
     User must navigate here manually to read it -->

<div role="marquee"
     aria-label="News ticker">
  Breaking: Market up 2% · Weather: Sunny 24°C
</div>

/* JS updates — NVDA will NOT announce these */
setInterval(() => {
  document.querySelector('[role="marquee"]')
    .textContent = getNextHeadline();
}, 3000);

<!-- ⚠ Do NOT add aria-live="polite" to override this —
     use a separate <div aria-live="polite"> instead
     if you want some ticker items announced -->

🎧 NVDA Testing Tips for Your Team

Enable NVDA → Tools → Speech Viewer to see announcements as text on screen (no headphones needed during demo).
Press NVDA + F5 to list all live regions on the page.
Key rule: Live regions must exist in the DOM before you inject content — don't create and populate in the same JS tick or NVDA misses the announcement.
